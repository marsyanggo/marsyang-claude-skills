# Save — Session Progress Recorder

Save this session's progress with full detail: auto-detect completed tasks, capture unlisted work, and write a structured journal entry to `SAVE.md`.

## How to behave

- Both files live at `<project-root>/` — use the current working directory as the project root
- Read both files before doing anything
- Be conversational but efficient — ask in one go, not one question at a time
- After writing, confirm what was saved

## Workflow

### Step 1 — Read both files in parallel

Read `TARGET.md` and `SAVE.md` simultaneously.

### Step 2 — Auto-detect completed tasks (diff)

From `TARGET.md`: collect all `[x]` tasks (with parent Goal name).
From last SAVE.md entry: collect tasks listed under `### 完成的 tasks`.
Diff = `[x]` tasks NOT in last SAVE.md entry = newly completed this session.

### Step 3 — Present diff + ask for additions in one message

Show the user everything at once:

> 「偵測到這次 session 新完成的 tasks：
> - 建立開發日誌  _(Goal: 法律合規準備)_
>
> **還有沒有這次做了、但還沒在 TARGET.md 的事？**
> （直接列出來，我會加進 TARGET.md 並標 [x]，例如：「設定 git / GPG / SSH」「建 GitHub repo」）
>
> **這次 session 的詳細紀錄：**（選填，有的話請描述，例如：用了什麼工具、踩了什麼坑、做了什麼決定）」

Wait for user's reply before proceeding.

### Step 4 — Update TARGET.md

For each task the user listed as "extra completed":
1. Find the most appropriate Goal section (fuzzy match by topic)
2. Append as `- [x] <task>` under that Goal's sub-tasks
3. If no good match, append under the most relevant Goal

Also mark any newly detected `[x]` items (already done in file, just confirming).

Update `_Last updated_` timestamp.

### Step 5 — Write SAVE.md

Prepend a new session entry at the **top** of `SAVE.md` (newest first), after the header block:

```markdown
## Session: YYYY-MM-DD HH:MM TZ

### 完成的 tasks
- <task>  _(Goal: <goal>)_
- <extra task added this session>  _(Goal: <goal>)_

### 詳細紀錄
<structured breakdown of what was done — tools used, decisions made, problems solved>
If user skipped this, write "無".

### 卡點 / 重要決定
<blockers encountered or key decisions made — extracted from user's notes>
If none, write "無".

### 下次從這裡繼續
<one concrete sentence: the exact next action to take, inferred from first incomplete task in active Goal>

---
```

If `SAVE.md` doesn't exist yet, create it with this header first:

```markdown
# Session Journal — ATF Validator

_Auto-generated by /save. Newest entries at the top._

---
```

### Step 6 — Confirm

Show the user a summary:
1. Tasks marked `[x]` in TARGET.md (auto-detected + newly added)
2. The full SAVE.md entry written

## Edge cases

- If no tasks at all: still write SAVE.md entry with notes
- If TARGET.md missing: warn and skip task update
- If SAVE.md has no prior entries: treat all current `[x]` as this session's work
- Never delete or overwrite existing SAVE.md entries — always prepend after header
